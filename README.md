# claude-dev-template

> Claude Code template with 8 specialized agents for building, reviewing, testing, and documenting full-stack apps.

A reusable Claude Code setup for full-stack projects. Drop it into any new project and get a complete agentic build system: separate backend and frontend builders, a reviewer agent, test runner, security auditor, performance analyzer, and docs writer вЂ” all governed by a single `CLAUDE.md` with a built-in self-improvement loop.

---

## What's Included

### Agents (`/.claude/agents/`)

| Agent | Role | Runs |
|---|---|---|
| `backend-builder` | Writes routes, controllers, models, middleware | Step 2 |
| `frontend-builder` | Writes components, pages, hooks | Step 4 |
| `reviewer` | Reviews each builder's output, scores and approves | Steps 3 & 5 |
| `test-runner` | Runs tests, fixes test code, surfaces blockers | Step 6 |
| `security-auditor` | Scans for vulnerabilities, exposed secrets, auth gaps | Step 7 |
| `performance-agent` | Identifies bottlenecks in backend and frontend | Step 8 |
| `docs-writer` | Synthesizes all agent output into `docs/architecture.md` | Step 9 |
| `debugger` | Traces root cause of bugs and writes regression tests | On demand |

### Skills (`/.claude/skills/`)

| Skill | Invocation | Purpose |
|---|---|---|
| `scaffold` | `/scaffold [app description]` | Generates the full folder structure for a new project |

### Commands (`/.claude/commands/`)

| Command | Invocation | Purpose |
|---|---|---|
| `init` | `/init` | Reads project state and prints a session brief at the start of each session |

```
        в†“

        в†“

        в†“

        в†“

```

The backend is always built before the frontend. The frontend builder reads the backend's API Contract Summary and only calls endpoints that exist. The reviewer runs twice вЂ” once per builder вЂ” with different checklists for each pass.

## Self-Improvement

Every agent is instructed to flag improvement opportunities at the end of its output. Proposals follow a structured format and require your explicit approval before any config file is changed. Agents cannot edit their own instructions, expand their own permissions, or modify the approval requirement.

Over time, your `CLAUDE.md` and agent files tighten up based on real friction from real builds.

## Getting Started

### 1. Clone or copy this template

```bash
git clone https://github.com/yourusername/claude-dev-template.git my-app
cd my-app
rm -rf .git && git init
```

### 2. Open Claude Code

```bash
claude
```

### 3. Scaffold your project

```
/scaffold A task management app with user auth and a dashboard
```

Claude will confirm the target directory, generate the full folder structure, create `.gitignore` and `.env.example`, and hand off to `backend-builder`.

### 4. Follow the build sequence

Each agent will signal when it's done and tell you which agent to invoke next. The full sequence typically runs in a single session.

## Stack

This template is **stack-agnostic**. No framework is assumed. When you start a new project, tell Claude which stack you're using and update the `# Tech Stack` section in `CLAUDE.md` before writing any code.

Tested patterns: Node/Express + React, Python/FastAPI + React, Next.js full-stack.
